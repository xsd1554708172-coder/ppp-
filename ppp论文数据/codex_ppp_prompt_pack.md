# PPP论文项目：Codex 启动与执行指令（基于当前全套正式文件包定制）

你现在接手的是一个**已经完成主要方法执行与正式结果生成**的论文项目。你的任务不是继续发散做新模型，而是**基于当前正式文件包，完成数据—结果—图表—写作口径的系统核查，并为正式正文写作提供稳定输入**。

## 一、你必须先理解的项目主题
论文主题为：

**《数字政府如何提升 PPP 项目治理质量——来自政务服务数字化改革与政策文本量化的经验证据》**

主线不是泛泛讨论数字经济，而是聚焦：
- 数字政府 / 政务服务数字化改革
- PPP 项目治理质量与推进结构
- 政策文本量化
- 项目级风险预警扩展

## 二、最重要的硬约束（任何时候不得违反）
1. **主模型只能是多期 DID / TWFE**  
   BERTopic、监督分类、项目级风险识别都只能辅助，不得抢主线。

2. **固定顺序不能打乱**  
   文本分析 → 变量构造 → 主识别 → 机制 → 异质性 → 稳健性 → 扩展分析。

3. **不能过度解释**  
   - 事件研究若 pretrend 显著，不得写“平行趋势成立”。
   - 机制若链条不稳，不得写“完整强中介链条成立”。
   - 异质性不强，不得写成第二主结论。
   - 第9部分不得写成全文主识别。

4. **一切以正式结果实体文件为准**  
   不得只根据 README、历史说明、旧版说明或口头交接复述结论。

5. **不准混入历史包/过程包/旧版结果**  
   `99_历史说明与证据_勿引用/` 仅用于冲突审计，不能直接当正文正式引用层。

6. **图表必须服务论证，而不是为了炫技而炫技**  
   但图表仍需高信息密度、体现工作量、有机器学习影子。

## 六、本轮任务不是重做模型，而是做“保险式系统核查”
请使用 Plan mode 先给出计划，再执行。执行时按以下六大任务推进：

### 任务 A：正式文件归类与排他性确认
- 识别哪些是正式主层文件

- 输出“正式文件清单”和“禁止直接引用文件清单”

### 任务 B：变量、层级、口径核查
重点核查：
- `did_intensity` / `did_any` / `treat_share`
- `ppp_quality_zindex`
- `ppp_quality_pca` / `ppp_quality_pca_rebuilt`

- `risk_struct_clean`
- `risk_text_count`

必须判断是否存在：
- 同名不同义
- 方向不一致
- 文档级与省—年级混用
- 历史版与正式版混用
- 标签定义与特征定义混淆
