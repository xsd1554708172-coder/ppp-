# Codex 网页版修改提示词

最后更新：2026-04-19

本文件提供两份可直接复制到 Codex 网页版中的提示词：

- `v1c修改建议 -> v1d`
- `v2c修改建议 -> v2d`

共同规则：

- 先读 `AGENTS.md`
- 先读 `00_环境说明与索引/`
- 先读 `ppp论文数据/` 中的正式口径文件
- 不要默认读“最新稿件”
- 必须先从 `修改建议/<token>修改建议/` 中找“深度研究报告明确对应的被审核稿件”
- 再去找那一份被审核稿件本体
- 修改后生成新的下一版本：
  - `v1c -> v1d`
  - `v2c -> v2d`
- 修改完成后：
  - 写说明文件
  - 写 verification 文件
  - 写 delivery note
  - 生成 zip
  - 写一份单独操作日志到 `修改稿/操作日志/`
  - 刷新 `修改稿/索引/`
  - 留底到对应 `修改稿留底/<token>/`

---

## 一、v1c 修改建议 -> v1d

```text
你现在直接工作在 GitHub 仓库：

`xsd1554708172-coder/ppp-`

本轮任务只处理 `v1c` 的修改建议，并在完成后产出 `v1d` 版本。

====================
一、先读取这些文件
====================

先读：

- `AGENTS.md`
- `00_环境说明与索引/01_工作区归类说明_20260419.md`
- `00_环境说明与索引/02_文件定位清单_20260419.md`
- `00_环境说明与索引/03_Git与运行环境同步说明_20260419.md`

再读 source-of-truth：

- `ppp论文数据/PPP_v3结果层状态表_20260413_1345.xlsx`
- `ppp论文数据/PPP_变量与模型最终采用口径表_20260413_1345.xlsx`
- `数据总表（一切数据基础）.xlsx`
- `ppp论文数据/参考文献/`
- `PPP_empirical_reinforcement_bundle_20260416_unified_v3/04_manuscript_integration/`

====================
二、版本识别规则
====================

本轮不要默认读取最新稿件。

先递归扫描：

- `修改稿/v1修改建议/v1c修改建议/`

优先识别其中的深度研究报告、审核报告、revision report、review 文件。

你必须先从这些建议文件中提取：

1. 被审核的具体稿件文件名
2. 若无完整文件名，则提取版本 token、日期、对象保留版/codexrev/本轮修订版等后缀
3. 明确本报告到底审阅的是哪一份 `v1c` 稿件

然后按这个顺序找原稿：

1. `修改稿/v1初始版/`
2. `修改稿/v1修改稿留底/v1c/`
3. `修改稿/v1说明文件/v1c/`
4. 若仍找不到，再查全仓库中与 `v1c` 有关的历史整合稿

默认以“被审核的那一版 v1c 稿件”为修改基底。
不能直接跳去改别的更新稿，除非你明确说明发生了“建议前推”。

====================
三、任务目标
====================

你必须：

1. 读取 `v1c修改建议` 文件夹中的建议
2. 识别并读取被审核的 `v1c` 原稿
3. 基于这些建议实际修改文章
4. 修改完成后输出 `v1d` 版本，而不是继续覆盖 `v1c`
5. 如建议涉及数据、变量、图表、回归、样本、稳健性、附录、文献或格式，必须实际处理
6. 如有必要，修正数据、处理逻辑、脚本并重跑
7. 同步正文、表、图、附录、说明文件中的数值和表述
8. 写说明文件、verification 文件、delivery note
9. 生成 zip
10. 将新稿归档到 `修改稿/v1修改稿留底/v1d/`
11. 新增一份操作日志到 `修改稿/操作日志/`
12. 刷新 `修改稿/索引/`
13. 提交仓库变更

====================
四、硬约束
====================

- 唯一主识别仍然是 `treat_share` 多期 DID / TWFE
- 不得把 trend-adjusted DID、leave-one-province-out、wild bootstrap、PSM-DID、DML、IV candidate、ML 扩展写成主识别
- 不得写 event study 证明平行趋势成立
- 不得抬高不稳健的 quality 结果
- 不得把机器学习写成因果识别
- 必须维持 PPP推进结构改善优先 这条主线
- 不得编造任何系数、显著性、机制、文献细节、图表结果或 rerun 结果

====================
五、输出要求
====================

在：

- `修改稿/v1说明文件/v1d/`

下至少生成：

1. `00_input_mapping.md`
2. `01_revision_tasklist.md`
3. `02_verification_report.md`
4. `03_delivery_note.md`
5. `v1d_codex_revision_bundle_<MMDD_HHMM>.zip`

并把新的稿件输出为：

- `修改稿/v1说明文件/v1d/PPP论文_完整论文初稿_公共管理风格_顶刊冲刺整合稿_v1d_<你的后缀>.docx`
  或对应可审读格式

归档到：

- `修改稿/v1修改稿留底/v1d/`

日志要求：

- 新增一份单独日志文件到 `修改稿/操作日志/`
- 文件名精确到秒
- 不覆盖旧日志

索引要求：

- 刷新 `修改稿/索引/修改稿索引总览.md`
- 刷新 `修改稿/索引/v1修改稿索引.md`

====================
六、最终回复必须包含
====================

1. 读了哪份 `v1c` 建议文件
2. 被审核的原始 `v1c` 稿件是哪一份
3. 是否直接基于该 `v1c` 修改，还是做了前推
4. 产出的 `v1d` 文件路径
5. 留底路径
6. 操作日志路径
7. 说明文件路径
8. zip 路径
9. 是否改了数据
10. 是否重跑
11. commit / branch / PR 信息
12. 剩余风险

现在直接执行，不要停在建议层。
```
