# v2e Literature Integration Patch

本文件基于本地 `v2d_DID_冲刺修订报告` 对 14 篇 DID 文献的整理，给出第2章与第4章的嵌入式文献 patch。未逐篇全文核验页码、固定效应和所有控制变量，因此不得编造页码或技术细节。未重跑，不得当作结果。

## 1. 文献使用原则

这 14 篇文献对本文最有价值的不是“都使用 DID”，而是提供了四类可迁移的审稿防守动作：

1. 原生处理名单：处理来自政策批次、行政批复、司法试点、基础设施落地或制度改革名录。
2. 外生政策时点：处理时点可由公告、试点批次、建设落地或制度实施年份审计。
3. 结果变量与处理变量分源：结果多来自企业、土地、人口、项目或市场交易数据库，而非与处理同源的文本工程。
4. 分层书写：主识别、防守检验、边界诊断和机制解释分开，不把稳健性检验写成第二主模型。

## 2. 第2章建议新增小节：制度冲击 DID 的证据链要求

### 建议标题

`2.x 制度冲击、处理名录与DID证据链`

### 可嵌入正文

制度冲击型 DID 研究的说服力，首先来自处理变量的可审计性，而不是估计量数量本身。已有研究围绕国家级新区、新城新区、国家级大数据综合试验区、行政诉讼跨区划管辖、社保征管体制改革、三峡送电、破产审理方式改革试点、数字产业集群政策等制度或基础设施变化展开，通常都能够指出处理组、对照组和处理时点的制度来源。相较于从研究者自建文本变量中直接推导处理强度，这类研究的共同优势在于处理名单具有外部制度落点，结果变量又多来自土地交易、企业经营、人口流动、创新、出口或就业等独立数据源。对本文而言，这一文献脉络提示：政务服务数字化改革影响 PPP 推进结构的经验识别，不能只停留在省年 `treat_share` 的最终数值上，而需要进一步公开其城市级处理名单、首次处理年份和省年聚合规则。

### 引用嵌入建议

- 原生城市/地区处理名单：新城新区建设、数字产业集群政策、行政诉讼跨区划管辖。
- 外生政策或基础设施时点：三峡送电、国家级大数据综合试验区、国家级新区。
- 分源机制与结果：智算中心与企业 TFP、行政区划改革与土地交易、社保征管改革与人口迁移。
- reduced-form 与机制分层：统一市场准入与企业创新行为。

## 3. 第4章建议新增说明：本文与 DID 文献的差异化定位

### 插入位置

第4.4节主识别模型说明之后。

### 可嵌入正文

需要说明的是，本文并不把 `treat_share` 处理为传统二元政策处理变量，而是将其界定为由城市级改革覆盖状态聚合而来的省年数字化改革暴露强度。与直接基于国家级新区设立、新城新区开建、司法改革试点、三峡送电或国家级大数据综合试验区批次构造处理名单的 DID 研究相比，本文的处理变量多了一层从城市到省年的聚合环节。因此，本文在解释上更强调平均条件关联和推进结构重组，而不是单一处理单元的标准 ATT。v2e 的补强方向，是把底层 city-year registry、首次处理年份、聚合权重和省年 `treat_share` 审计表公开出来，以缩短本文与原生处理名单型 DID 研究之间的证据链差距。
