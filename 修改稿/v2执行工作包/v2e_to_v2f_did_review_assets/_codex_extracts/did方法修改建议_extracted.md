第一部分：关于 PPP 文本研究是否适合 DID 的判断（整理稿）

一、结论先行：普通 DID 不宜作为主模型。因为 PPP 2014 推广政策、2017 规范清理政策、2023 新机制等都属于全国性政策，不存在天然的未处理组。与压缩包中那些试点改革、司法改革、基础设施送电、国家级新区设立等典型 DID 论文相比，你原始的政策冲击—处理组—对照组结构并不完整。

二、如果一定要用 DID，最可行的是强度 DID，而不是普通二元 DID。最有识别价值的冲击是 2017 年财政部《关于规范政府和社会资本合作（PPP）综合信息平台项目库管理的通知》，因为它明确针对 PPP 泛化滥用、项目库管理和隐性债务风险，是更清晰的制度转向。相对于此，全国统一政策下真正可利用的横向差异来自各地区政策前 PPP 暴露强弱，因此应构造事前 PPP 项目规模/投资额/财政承诺占比 × 政策后这一强度交互项。

三、你的文本数据可以作为 DID 的结果变量，但前提是先面板化。也就是说，BERTopic、GNN、政策立场网络、风险治理主题占比等变量不能直接拿文档级数据回归，而要先聚合为省—年、市—年或地区—季度面板。例如，RiskRegulation、CentralSimilarity、StanceScore 等可以作为因变量 Y。

四、DID 在这篇 PPP 论文中最多补强‘制度语义响应’而非‘PPP 实体绩效改善’。如果 Y 是风险监管主题占比、中央政策相似度、政策立场得分，那识别的是中央规范政策是否导致地方政策文本更明显地转向规范、监管和财政约束语义，而不能直接推导出 PPP 项目绩效或公共服务质量改善。

五、在没有补齐外部数据的情况下，不应强行使用 DID。若目前只有政策文本数据，最适合的方法反而是结构突变检验、中断时间序列、动态主题模型、事件窗口文本响应和语义网络演化，而不是形式上更复杂但识别基础不足的 DID。

第三部分：从前两轮答复整理出的直接建议

第一，不要为了增加方法复杂度而强行使用 DID。识别基础如果不硬，额外的方法名不会增强论文，反而会引来更严格的质疑。

第二，主方法与补充识别要分层。文本挖掘负责制度测量，DID 负责在合适的外生冲击下检验差异化响应，二者不能混成一套逻辑。

第三，政策冲击必须回到真实制度文本、真实处理名单和真实时点，而不是从结果变量附近再发明一个处理变量。

第四，任何 DID 结论都必须与被解释变量的含义严格对应。文本指标只能说明制度表达变化，不能越界解释为项目绩效改善。

第五，如果未来能补齐项目库、财政、地区发展与城市级改革名单，则 DID 可以作为补充识别显著增强论文的说服力；如果不能补齐，就应果断改用更匹配数据结构的方法。

附：前两轮答复的主线逻辑

主线一：不是所有有政策背景的研究都适合 DID。必须同时具备清晰冲击、可区分处理/控制、可区分前后和可检验的平行趋势。

主线二：全国统一政策不等于不能做 DID，但更适合强度 DID，而不是普通二元 DID。

主线三：文本挖掘可以为 DID 构造结果变量或机制变量，但文本结果必须先被面板化，且不能越界解释。

主线四：现代 DID 的关键不只是‘回归一个交互项’，还包括事件研究、现代稳健性估计、安慰剂和同期政策排除。

主线五：如果数据结构与研究问题不匹配，最优解不是强行上 DID，而是换用更合适的方法。先修正两点

三、为什么我推荐 2017 年 PPP 规范清理政策

这个判断不是来自压缩包文章，而是来自 PPP 官方政策文件 + DID 识别逻辑。

1. 2014 年 PPP 推广政策不适合作为主 DID 冲击

2014 年财政部发布《关于推广运用政府和社会资本合作模式有关问题的通知》，目标是推广 PPP、拓宽城镇化融资渠道、促进政府职能转变、形成有利于 PPP 发展的制度体系。它是制度扩张起点，但全国同步实施，且你若样本主要从 2014 年开始，政策前数据不足。(中国政府网)

所以它适合作为制度背景，不适合作为 DID 主冲击。

2. 2017 年财办金〔2017〕92 号更适合作为制度转向冲击

财政部 2017 年《关于规范政府和社会资本合作（PPP）综合信息平台项目库管理的通知》明确提出，要规范 PPP 项目运作，防止 PPP 异化为新的融资平台，遏制隐性债务风险增量，并纠正 PPP 泛化滥用现象。这个政策目标与“风险监管、财政承受、项目库管理、规范发展”等文本语义高度相关。(jrs.mof.gov.cn)

因此，我推荐它作为你论文的 DID/事件研究冲击点。

但由于这是全国性政策，所以我没有建议普通 DID，而是建议：

[Y_{it}=\alpha+\beta(Exposure_i^{pre}\times Post_t)+\gamma X_{i,t-1}+\mu_i+\lambda_t+\varepsilon_{it}]

即 强度 DID。

3. 2023 年 PPP 新机制可作为补充事件，不宜直接作为主 DID

2023 年国务院办公厅转发《关于规范实施政府和社会资本合作新机制的指导意见》，明确新机制聚焦使用者付费项目、全部采取特许经营模式、优先选择民营企业参与，并要求规范推进建设实施和运营监管。(中国政府网)

但如果你的样本截至 2023 或 2024，政策后观测太短，不适合做主 DID。它更适合作为事件描述、结构突变分析或补充事件研究。

五、我上轮结论的证据链条

结论 1：你的文章不适合普通 DID

依据：压缩包中使用普通或交错 DID 的论文都具有清晰的处理组与控制组，例如数字产业集群试点、大数据综合试验区、破产司法改革试点、行政诉讼跨区划管辖改革、国家级新区设立等。

你的情况：PPP 中央政策是全国性政策。所有地方政府原则上都受到影响，不能自然划分“被处理地区”和“未处理地区”。

所以：普通 DID 不稳。

结论 2：若使用 DID，应使用强度 DID

依据：《以链养税》处理的是全国性财政制度改革，但利用各地财政激励强度差异构造强度 DID。

你的情况：PPP 规范清理政策全国实施，但各地政策前 PPP 项目规模、PPP 投资额、PPP 财政承诺、PPP 政策密度不同。

所以：可以构造：

[PPPExposure_i^{pre}\times Post_t]

而不是简单的：

[Treat_i\times Post_t]

结论 3：DID 只能补强你的“政策文本响应”，不能证明 PPP 实体绩效

依据：DID 应用论文的 Y 都与政策目标直接相连。比如三峡送电影响企业出口，破产司法改革影响企业雇佣，行政诉讼管辖改革影响社保遵从。

你的情况：如果你的 Y 是 BERTopic/GNN 生成的语义指标，则它衡量的是地方政策文本变化，不是项目绩效、财政风险真实变化或公共服务供给质量。

所以：结论应限定为：

中央 PPP 规范清理政策是否引起地方 PPP 政策文本语义的差异化响应。

不能写成：

中央 PPP 政策显著改善了 PPP 治理绩效。

结论 4：如果数据还没有面板结构，暂时不应 DID

依据：压缩包中的 DID 论文全部具有单位—时间数据结构。即使《创新质量信号》使用文本嵌入，也仍然把数据拓展为专利—年份面板。

你的情况：如果 GitHub 中只有文档级政策文本，而没有地区—时间面板和 PPP 暴露度，就不能直接估计 DID。

所以：需要先构造：

[地区/城市/省份 \times 年份/季度/月度]

面板。
