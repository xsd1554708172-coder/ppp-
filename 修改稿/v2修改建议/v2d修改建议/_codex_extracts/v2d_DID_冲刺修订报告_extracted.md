## P138

这些文献为什么比一般 DID 更硬

## P139

这些文献之所以让人感觉“更经得起质疑”，核心并不在于它们使用了更多名词，而在于它们大多遵循了一个很硬的识别顺序：

## P140

flowchart TD    A[外部制度/司法/基础设施冲击] --> B[明确处理名单与首次处理时点]    B --> C[与结果变量同层级的单位面板]    C --> D[单位固定效应 + 年份固定效应]    D --> E[动态诊断 / 平行趋势 / 安慰剂 / 异质性]    E --> F[独立来源的机制变量]    F --> G[主识别、主防守、边界诊断分层书写]

## P141

第一，它们的处理变量不是“从结果附近倒推出来”的。国家级新区、跨区划管辖改革、负面清单、试验区、三峡送电、社保征管体制改革等，都是可以在官方制度文本、名录、批次中被核对的外部冲击。研究者做的是“匹配”和“映射”，不是“再发明一个处理”。这点与您当前 treat_share 由城市处理状态再向上聚合到省年、且城市级原始名单尚未完整显式落地，形成鲜明对比。[3] [4]

## P142

第二，它们的结果变量往往与处理变量分源。例如，行政诉讼改革研究看企业社保遵从，智算中心研究看企业 TFP，撤县设区研究看土地交易，社保征管改革研究看迁移与非正规就业，三峡送电研究看出口规模。换言之，处理来自制度，结果来自企业、地块、个体或市场交易数据库，数据链天然更硬。您的论文则把 treat_share 与 A/B/C/D 都建立在政策文本工程之上，虽然 v2d 已经诚实地把后者降为“证据线索”，但这种同源性依然是根本弱点。[30] [6]

## P143

第三，它们通常把主识别、防守检验、机制分析、异质性分析分开写，而不是混成一锅。您仓库里最新的 v2d 已经开始沿这个方向走：treat_share 多期 DID/TWFE 是唯一主识别，trend-adjusted DID 与 wild cluster 是防守层，A/B/C/D 是文本证据线索，项目级模型是治理辅助识别。这个方向是对的，说明您并不是方法上完全偏离，而是还没把“数据层和复现层”的那一步补硬。[7] [4] [6]
