确保以下两件事同时成立：

1. 每次本地修改完成后，GitHub 仓库同步更新。
2. GitHub 仓库有新内容时，本地运行环境也同步到最新状态。

## 建议操作顺序

### 开始一轮新任务前

先同步本地环境：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\sync_from_github.ps1"
```

作用：

- `git fetch origin`
- `git pull --rebase origin main`
- 保证本地运行环境先对齐 GitHub 最新内容

如果你想先检测远端是否有更新、并且只在工作区干净时才自动同步，执行：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\check_remote_signal_and_sync.ps1"
```

作用：

- `git fetch origin`
- 检查 `origin/main` 与本地 `HEAD` 是否不同
- 检查 `REMOTE_SYNC_SIGNAL.json` 的 signal sequence 是否变化
- 只有在本地工作区干净时才执行真正同步
- 若本地有未提交改动，则只记录状态，不强拉远端

如果你想持续轮询，可以运行：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\watch_remote_sync.ps1" -IntervalSeconds 300
```

作用：

- 每 300 秒检查一次远端
- 把检查结果写入：
  - `C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\00_环境说明与索引\REMOTE_SYNC_POLL.log`

## Windows 计划任务自动同步

如果你希望“不打开 Codex 也能自动同步”，当前已经支持 Windows 计划任务方案。

注册脚本：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\register_windows_remote_sync_tasks.ps1" -IntervalMinutes 5
```

会配置两类自动触发：

- Windows 周期任务：`Codex_PPP_RemoteSync_Every5Min`
  - 每 5 分钟检查一次远端同步。
- Startup 自启快捷方式：
  - `C:\Users\陈楚玲\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\Codex_PPP_RemoteSync_OnStartup.lnk`
  - 每次登录 Windows 后立即检查一次远端同步。

彻底卸载脚本（会删除 Startup 自启快捷方式，不用于日常关闭）：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\unregister_windows_remote_sync_tasks.ps1" -IntervalMinutes 5
```

手动停止脚本（临时关闭，不删除 Startup 自启快捷方式）：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\stop_windows_auto_sync.ps1" -IntervalMinutes 5 -StopRunningProcesses
```

作用：

- 关闭自动同步开关
- 禁用每 5 分钟周期任务
- 保留 Startup 自启快捷方式
- 可选终止当前已经启动的自动同步相关进程

统一开关脚本（日常开启/关闭都优先用这个）：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\set_windows_auto_sync.ps1" -Mode Enable -IntervalMinutes 5
```

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\set_windows_auto_sync.ps1" -Mode Disable -IntervalMinutes 5 -StopRunningProcesses
```

作用：

- `Enable`：启用每 5 分钟周期任务，并保留 Startup 自启快捷方式
- `Disable`：禁用周期任务，但保留 Startup 自启快捷方式
- Startup 触发时会先读取 `AUTO_SYNC_CONTROL.json`，若已关闭则直接退出，不执行同步

说明：

- 计划任务调用的是：
  - `C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\run_hidden_sync_check.cmd`
- Startup 自启快捷方式也指向同一个隐藏启动包装脚本。
- 实际执行的仍是安全同步脚本：
  - `C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\check_remote_signal_and_sync.ps1`
- 如果本地有未提交改动，任务会跳过同步，不会强制覆盖本地文件。

### 完成一轮任务后

如果本轮有 `v1*` / `v2*` 成稿，需要优先用一键收尾脚本：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\finalize_revision_task.ps1" -SourcePath "成稿路径" -Token "v1a"
```

作用：

1. 把成稿复制到对应留底目录。
2. 按 `<token>_<MMDD_HHMM>.<ext>` 命名。
3. 自动执行 Git 提交与推送。

如果本轮没有新的 `v1*` / `v2*` 成稿，但有其他工作区改动，则执行：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\陈楚玲\Desktop\ppp论文数据\codex项目\修改稿\scripts\git_sync_workspace.ps1" -CommitMessage "说明本轮改动"
```

## 脚本说明

- `sync_from_github.ps1`
  - 用于“开始工作前”拉取远端最新内容。
- `check_remote_signal_and_sync.ps1`
  - 用于“检查远端变化并在安全前提下自动同步”。
- `guarded_sync_check.ps1`
  - 用于“先读取本地自动同步开关，再决定是否真正执行同步检查”。
- `watch_remote_sync.ps1`
  - 用于“持续轮询远端变化”。
- `set_windows_auto_sync.ps1`
  - 用于“统一控制 Windows 自动同步的开启与关闭，并保留 Startup 自启快捷方式”。
- `stop_windows_auto_sync.ps1`
  - 用于“调用统一开关脚本切到 `Disable`，并可选停止当前相关进程”。
- `git_sync_workspace.ps1`
  - 用于“完成修改后”提交、同步远端、再推送。
- `finalize_revision_task.ps1`
  - 用于“成稿交付级修改”的留底 + Git 收尾。

## 注意事项

- 如果本地有未提交改动，`sync_from_github.ps1` 会拒绝直接拉取，避免把未提交工作冲掉。
- 如果本地有未提交改动，`check_remote_signal_and_sync.ps1` 也会拒绝自动同步，并把状态写入 `LOCAL_SYNC_STATE.json`。
- 如果 `AUTO_SYNC_CONTROL.json` 记录为 `enabled=false`，Startup 触发和隐藏触发都会直接退出，不执行同步。
- 如需彻底移除自动同步入口，才使用 `unregister_windows_remote_sync_tasks.ps1`；日常关闭一律用 `set_windows_auto_sync.ps1 -Mode Disable` 或 `stop_windows_auto_sync.ps1`。
- 如果远端和本地同时发生变更，`git_sync_workspace.ps1` 会先拉取并 rebase，再推送。
- 如果发生冲突，不要强推，先在本地解决冲突后再继续。
- `REMOTE_SYNC_SIGNAL.json` 是 Git 跟踪文件；`LOCAL_SYNC_STATE.json`、`REMOTE_SYNC_POLL.log` 和 `AUTO_SYNC_CONTROL.json` 是本地文件，不进入 Git。
