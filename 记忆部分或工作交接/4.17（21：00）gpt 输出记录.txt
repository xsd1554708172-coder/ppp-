# 第三部分：项目执行过程中的经验、教训与关键决策点

## 1. 你强调过的核心要求

这些是新窗口必须继承的高优先级原则：

第一，**真实数据优先**。不能没跑数据就先写结果，不能脱离正式文件空想。
第二，**不能把弱结果写成强结论**。尤其是不能把不显著质量结果、薄弱机制、弱异质性、降格补充识别写满。
第三，**不能让补充识别喧宾夺主**。`stack DID`、`cohort ATT`、动态补充识别都不能抬高到主识别替代层。
第四，**不能把机器学习第9部分写成炫技主亮点**。它只能是扩展分析。
第五，**文本分析必须服务于治理测度与变量工程，而不是变成主模型表演。**
第六，**图表—正文—证据必须一一对应**，不能只停留在“判断层面”不动文件。

## 2. 我曾经犯过的错误，以及你如何纠正

根据 4.16 交接文本与本窗口实际推进过程，可明确总结以下几类错误：

第一类错误：**只做判断，不真正动文件。** 你明确指出不能只说“有没有问题”，而必须真的改表、导出结果、生成包。后面三项 DID 补强之所以会真正成型，就是在这个压力下做出来的。

第二类错误：**图表—正文—证据错配。** 早期曾出现表题与表内容错配、图表数量和作用不匹配、机制与主文接口不顺等问题。后续才形成了“先对齐图表—正文—证据，再导出补强包”的纪律。

第三类错误：**把补充识别写得过满。** 比如 stack DID 一度差点被写成“主结论不依赖 TWFE”，cohort ATT 一度差点被写成“更强稳健补强”，但你反复要求根据真实结果降格。

第四类错误：**混淆“结果包交付齐全”和“补强成功”。** 你明确指出：诚实降格 ≠ 增强成功。也就是说，“做了很多方法”不等于“识别更强了”。

第五类错误：**工程交付不规范。** 包括命名乱码、脚本绝对路径、压缩包结构错误、bundle 路径坏掉等。这一点在 v2/v3 Codex 结果包审查中非常明显；后来短名包才修复了 zip 路径结构。

## 3. 遇到过哪些困难

最核心困难有四类：

第一，**样本结构困难**。正式主面板是省级面板，样本量不大；处理时点高度集中（2016 为主，2017 极少）；never-treated 很小。这直接限制了 stack DID / cohort ATT 的发挥空间。

第二，**DID 前趋势风险**。当前正文自己就承认处理前趋势不完全平稳，导致事件研究只能用于路径展示与边界说明。这是整个项目最核心的识别难点。

第三，**补充识别的高低回报判断**。

* stack DID：怎么调都只是方向一致，显著性不强；
* cohort ATT：看似更强，但一旦 leave-one-cohort-out 就暴露 2017 cohort 依赖；
* 动态补充识别：救不起来前趋势，只能附录。
  因此关键不是“还能不能再做”，而是“值不值得继续投精力”。

第四，**工程与版本管理困难**。
压缩包命名、乱码、路径、结果包和正文替换文本多轮迭代、旧版与新版口径并存，这些都让版本管理变复杂。这里的新窗口必须非常警惕：**凡有 0912、1048、0256、v2、v3、clean、unified_v2、short v3 等多个口径并存时，一律以当前正文主版本 + 正式 1048 结果 + 已核查 patch 为准。**

## 4. 关键决策点

### 为什么坚持 `treat_share` 多期 DID/TWFE 作为唯一主识别？

因为：
当前正式主面板、理论框架、正文结构、主结果表、摘要/结论都围绕它展开；其他方法均未形成更强替代。这不是偏好问题，而是**已有结果形态**决定的。

### 为什么 stack DID 最终只能降格？

因为 round2/round3 结果始终只做到**方向一致但显著性偏弱**，不足以写成“显著增强主识别”。

### 为什么 cohort ATT 只能作为探索性替代估计？

因为 leave-one-cohort-out 已经证明，它的“更强部分”主要被 2017 单一 cohort 拉出来，不稳。

### 为什么动态补充识别只能放附录？

因为它没有改善前趋势风险，只能防止误用，不能加分。

### 为什么补强方向后来转向趋势调整 DID、leave-one-province-out、小样本稳健推断？

因为在现有数据结构下，这三项更直接回答三个最关键的质疑：

* 是不是省份原有趋势在驱动结果；
* 是不是单一省份在驱动结果；
* 显著性是否过度依赖常规聚类推断。
  这三者是当前数据约束下的高回报补强。

## 5. 最终吸取的经验

在现有数据约束下，高回报补强就是：**趋势调整 DID、leave-one-province-out、小样本稳健推断**。
看起来高级但回报很低的，是：继续筛 stack DID 规格、继续包装 cohort ATT、继续救动态补充识别、继续强调 never-treated vs not-yet-treated 的细枝末节。
