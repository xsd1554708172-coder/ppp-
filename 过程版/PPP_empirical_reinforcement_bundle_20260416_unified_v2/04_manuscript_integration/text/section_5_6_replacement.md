# 5.6 稳健性检验与主识别防守（替换稿）

本节并不试图通过不断叠加新方法来改写表4中的主识别结果，而是围绕当前最稳的 `treat_share` 多期 DID / TWFE 结论，检验“推进结构改善优先”这一核心判断在几个关键识别脆弱点下是否仍然成立。换言之，本节的任务不是进行 method stacking，而是针对共同趋势设定、单一地区驱动风险和小样本推断偏乐观等具体质疑，依次给出防守性证据，并明确各层结果的解释边界。

## 5.6.1 趋势调整型防守检验

首先，为回应事件研究中处理前系数并非完全平稳这一识别顾虑，本文在基准规格上进一步加入省份线性时间趋势，对核心结果进行趋势调整型 DID 检验。该检验保持正式主面板、处理变量、控制变量以及地区固定效应和年份固定效应口径不变，仅额外控制省份层面可能存在的差异化线性趋势，因此它的性质是防守性稳健性检验，而不是新的主模型，也不替代表4中的主识别结果。

趋势调整后的结果显示，`proc_share` 仍保持显著负向，说明采购阶段占比下降这一核心判断并不完全依赖于共同趋势设定；`exec_share` 的估计方向仍为正，但统计强度明显减弱，表明执行阶段占比上升这一结论在方向上较稳，但对趋势设定更为敏感。相比之下，`ppp_quality_zindex` 在趋势调整后不再维持基准回归中的正向估计，且统计上继续不显著，因此质量型口径更不能承担全文主结论。由此可见，趋势调整型检验增强的是主识别的防守能力，而不是把全部结果一并抬升为同等强度的稳健结论。

## 5.6.2 单一地区驱动风险诊断：leave-one-province-out

其次，本文实施 leave-one-province-out / jackknife 诊断，以判断基准 DID 结果是否主要由个别地区驱动。在保持处理变量、控制变量、固定效应和地区层聚类标准误口径不变的前提下，本文依次剔除一个省份并重复估计基准规格。结果表明，`exec_share` 与 `proc_share` 在删除任一省份后均未出现方向翻转，说明推进结构主结论并非由单一省份机械驱动。与此同时，个别省份删除后两项推进结构结果的显著性均有所减弱，这意味着主结论在方向上较稳，但统计强度对少数地区仍具有一定敏感性。对 `ppp_quality_zindex` 而言，leave-one-province-out 继续表现出幅度波动较大、统计支持不足的特征，因此它仍应被理解为边界性结果，而非主结论。

## 5.6.3 更保守推断下的边界说明

再次，考虑到当前样本为省级面板、地区聚类数量有限，本文进一步将 wild cluster bootstrap 作为更保守的小样本推断层。这里的重点并不是把 bootstrap 结果写成新的强化证据，而是检验核心推进结构结果在更保守推断下是否仍保持方向稳定。结果显示，`exec_share` 与 `proc_share` 的方向均与基准 DID 一致，但 bootstrap p 值明显高于常规聚类推断，这说明推进结构结论在更保守的小样本推断下保留了方向稳定性，但统计强度更为敏感。相较之下，`ppp_quality_zindex` 在更保守的小样本推断下同样未形成稳定支持，因此不宜上升为主结论。该层检验仅提供边界说明，并不替代表4中的主识别结果。

## 5.6.4 其他补充识别与稳健性结果（简述）

除上述主防守层之外，本文还参考补充识别诊断结果，对处理时点异质性和替代聚合框架做了额外检查。现有 stack DID 结果表明，`exec_share` 与 `proc_share` 在处理进入框架下总体保持与基准 DID 一致的方向判断，但统计强度明显减弱，因此更适合被理解为方向性补充，而非对主识别的实质增强。更稳健的 cohort ATT 替代估计在全 cohort 聚合口径下曾对 `proc_share` 给出更强负向结果，但 leave-one-cohort-out 敏感性检验显示，该增强效果对 2017 单一 cohort 较为敏感，因此也不宜被解释为稳定补强。动态补充识别同样未能真正解决处理前趋势风险，最多只能保留为附录层的边界诊断。也就是说，stack DID、cohort ATT 与动态补充识别在本文中都只承担方向性补充或边界诊断功能，并不替代表4中的主识别结果。综合而言，补充识别的主要作用在于防止方向误判和补充边界诊断，而不是替代或显著强化表4中的主识别结论。
